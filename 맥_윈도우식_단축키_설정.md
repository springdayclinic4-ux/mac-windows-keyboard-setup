## 🔁 수정자 키 스왑 (맥북 내장 키보드 한정)

```
[Control] [Option] [Command] [Space] [Command(우)]
            ↓         ↓                      ↓
          Win키     Alt키                  한/영
```

| 실제 키 | 스왑 후 동작 | 적용 디바이스 |
|---|---|---|
| 좌측 **Option** | **Command (= Windows 키)** | **맥북 내장만** |
| 좌측 **Command** | **Option (= Alt 키)** | **맥북 내장만** |
| 우측 **Command** | **한/영 전환** (모든 키보드) | 전부 |
| Control | 그대로 (Ctrl 역할) | 전부 |

> 외부 키보드는 이미 Win 키가 스페이스 옆이라 스왑 불필요. 그대로 윈도우 배치.

## 🔌 디바이스 자동 적용 (중요)

**모든 신규 키보드/마우스에 자동 적용** 되도록 디바이스 식별자 일반화:

```json
"devices": [
  { "identifiers": { "is_pointing_device": true }, "modify_events": true },
  { "identifiers": { "is_keyboard": true }, "modify_events": true }
]
```

| 새 디바이스 종류 | 자동 적용 여부 |
|---|---|
| 외부 USB 키보드 | ✅ 즉시 |
| 블루투스 키보드 | ✅ 즉시 |
| 외장 마우스 | ✅ 즉시 |
| 매직 트랙패드 | ✅ 즉시 |

**디바이스 조건이 걸린 규칙**:
- 좌측 Option↔Cmd 스왑 → `is_built_in_keyboard: true` (맥북 내장만)
- 우측 Alt → 한/영 → `is_built_in_keyboard: false` (외부 키보드만)

## 🔧 트러블슈팅

### "전체가 안 먹힌다"
```bash
pgrep -l karabiner_grabber  # 결과 없으면 데몬 죽음
```
→ 드라이버 확장 미승인. 시스템 설정 → 개인정보 보호 및 보안 → 맨 아래 "허용" → 재부팅.

### "Option 단축키만 안 먹힌다"
→ Karabiner-Elements 앱 → Complex Modifications에서 "Windows 키 단축키" 규칙 활성화 확인.

### "외부 키보드가 안 먹힌다"
→ Karabiner-Elements → Devices 탭에서 해당 키보드 **"Modify events"** 토글 ON 확인. 또는 새 키보드 식별 마법사 통과.

### "특정 키가 비표준 신호를 보낸다" (예: HCT 키보드 End = left_command)
→ Karabiner-EventViewer 실행 → 키 누르고 `name` 확인 → 키보드 펌웨어 문제. 다른 키보드 사용 권장 또는 Mac/Win 모드 토글 확인.

### "한/영 전환이 안 된다"
→ 시스템 설정 → 키보드 → 입력 소스에 **"한국어 2벌식"** 추가돼 있는지 확인.
→ 입력 소스 ID가 `com.apple.inputmethod.Korean.2SetKorean` 인지 확인:
```bash
defaults read com.apple.HIToolbox AppleEnabledInputSources | grep -i korea
```

## 🎁 외부 프리셋 (커뮤니티 검증)

`rux616/karabiner-windows-mode` 의 **Windows Shortcuts** 프리셋(63개 규칙)을 Karabiner 자산 폴더에 import해둠.

- 출처: https://github.com/rux616/karabiner-windows-mode
- 파일 위치: `~/.config/karabiner/assets/complex_modifications/1777047574.json`
- 백업본: `karabiner_프리셋_windows_shortcuts.json` (이 저장소)

### 활성화 방법 (필요 시)
1. Karabiner-Elements 앱 열기
2. **Complex Modifications** 탭
3. 우측 하단 **Add predefined rule** 클릭
4. **"Windows Shortcuts"** 섹션에서 원하는 규칙 옆 **Enable**

### 현재 활성화 안 된 이유
이 저장소의 custom 규칙들이 이미 핵심 동작(Ctrl+T·C·V·F 등)을 커버하고 있어 중복 활성화는 충돌 위험. 다음 항목이 추가로 필요할 때만 enable:
- Dock 핀 앱 단축키 (`Cmd + 1~9`)
- 터미널 에뮬레이터 / IDE / 원격 데스크톱별 예외 처리
- Chrome 전용 단축키 (`F5` 새로고침 등)
